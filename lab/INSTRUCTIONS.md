## Two-Level Rule

- **Lab level (Git)**: `lab/`, `projects/`, `members/*/ME.md`, settings, journal, skills — all shared, all visible.
- **Personal level (Local)**: `ME.private.md` and API keys stay local. **Never commit ME.private.md**.

## Onboarding

New members: clone repo → `bash scripts/setup.sh <name>` → fill ME.md (including Sharing section) → PR → merge. Then read assigned projects' GOAL.md → PROGRESS.md to get started.
