## Roles

- **Admin** — Can modify lab/ config, GOAL.md, approve new members, promote skills to lab-wide, maintain MEMBERS.md
- **Member** — Can modify their own members/ directory and assigned projects/
- **Intern** — Same as Member, only participates in assigned projects

## Git Collaboration Rules

- All members are added as **Collaborators** on the GitHub repo (by Admin)
- `main` branch has branch protection (require 1 approval)
- New members submit their first commit via PR
- Routine updates (journal, progress) can be pushed directly to main after Admin approval

## How to Use

- **Agent**: When a user mentions someone, check this file for their role, projects, and expertise
- **Agent**: During new member onboarding, read this file to introduce the team
- **Admin**: After a new member joins, add a row (including Can Help With), commit + push
- **Member**: When responsibilities change or you have new expertise to share, ask Admin to update or submit a PR
