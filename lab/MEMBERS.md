## Roles

- **Admin** — 可修改 lab/ 配置、GOAL.md、审批新成员、审批 skill 提升到 lab-wide、维护 MEMBERS.md
- **Member** — 可修改自己的 members/ 目录和被分配的 projects/
- **Intern** — 同 Member，只参与被分配的项目

## Git 协作规则

- 所有成员作为 **Collaborator** 加入 GitHub repo（Admin 添加）
- `main` 分支有 branch protection（require 1 approval）
- 新成员首次提交通过 PR
- 日常小更新（journal、progress）经 Admin 允许后可直接 push main

## How to Use

- **Agent**: 用户提到某个人时，查此文件确认角色、负责项目和可对接内容
- **Agent**: 新成员 onboarding 时，读此文件介绍团队
- **Admin**: 新人加入后追加一行（含 Can Help With），commit + push
- **Member**: 职责变更或新增可分享内容时，请 Admin 更新或自行 PR
